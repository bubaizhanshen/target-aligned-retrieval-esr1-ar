# Security Policy

## Supported Scope

This repository contains research code, processed public-data artifacts, and
result tables. The main security concerns are dependency issues, unsafe file
handling in scripts, accidental credential disclosure, and data-provenance
mistakes.

## Reporting a Vulnerability

Please open a private security advisory on GitHub if the issue may expose
credentials, private data, or a vulnerability that should not be public before a
fix is available. For lower-risk dependency or documentation issues, open a
regular GitHub issue.

Reports should include:

- affected file or workflow
- reproduction steps
- expected impact
- suggested fix, if known

The maintainer will acknowledge credible reports and prioritize fixes that
affect reproducibility, data integrity, or user safety.
